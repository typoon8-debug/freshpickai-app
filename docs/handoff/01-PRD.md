# FreshPick — Product Requirements Document
*v1.0 · Mocha Mousse Edition*

## 1. One-liner
**우리가족을 위한 AI 큐레이팅 장보기** — 테마 카드(흑백요리사·제철 K-팜·홈시네마 등)로 한 끼를 고르고, AI에게 변형을 요청하고, 가족과 투표한 뒤, 재료를 한꺼번에 장바구니에 담아 새벽배송 받는 모바일 커머스.

## 2. 타깃 페르소나
| Persona | 나이 | 핵심 Pain | 핵심 Gain |
|---|---|---|---|
| **메인 큐레이터 (엄마)** | 35–45 | "오늘 뭐 먹지?"의 무한 반복, 마트앱 검색 피로 | AI가 가족 취향·예산·시간을 알아서 큐레이팅 |
| **주말 셰프 (아빠)** | 35–50 | 평소엔 무관심, 주말엔 폼나는 한 끼 도전 | 흑백요리사 재현 카드로 미션처럼 요리 |
| **Z세대 자녀 (서연 14)** | 12–18 | 부모 메뉴 = 지루함, SNS 트렌드 갈증 | 트렌드 카드 + 키즈/청소년 모드 + 투표 발언권 |
| **키즈 (하준 8)** | 5–10 | 채소 거부, 편식 | 마스코트·미션·업적으로 재미있게 한 입 도전 |

## 3. 핵심 가치 제안 (3 Whys)
1. **카드 = 의사결정 단축.** "검색"이 아닌 "펼침" — 80여 메뉴를 30초 안에 좁힘.
2. **AI = 변형 자유도.** 비건/매운맛/10분/8천원이하를 자연어로 요청하면 즉시 RAG 재추천.
3. **가족 = 함께 결정.** 투표·인기 TOP3·트렌딩으로 "가족 키친" 데이터를 쌓아가며 큐레이션 정확도 ↑.

## 5. 비기능 요구사항
- **성능:** Lighthouse Mobile 90+ · 초기 로딩 LCP < 2.5s
- **접근성:** WCAG AA · 모든 인터랙션 44px hit target
- **i18n:** 한국어 우선, 영어 stub
- **반응형:** 375px 모바일 최적화 (태블릿/데스크톱은 P2)
- **결제:** 카카오페이/네이버페이/카드/계좌이체
- **배송:** 샛별배송 (당일 23시 주문 → 익일 6시 도착)

## 7. 기술 스택 결정
| 영역 | 선택 | 이유 |
|---|---|---|
| Framework  | Next.js 14 App Router | RSC + Streaming · 한국 시장 최적화 |
| 언어       | TypeScript            | 타입 안전 + Claude Code 협업 효율 |
| 스타일     | Tailwind + shadcn/ui  | 토큰 기반 빠른 구현 |
| 상태       | Zustand               | Redux보다 가볍고 RSC 친화 |
| 데이터     | TanStack Query + RSC  | 서버 상태와 클라이언트 상태 분리 |
| AI         | Claude API (RAG 백엔드 별도) | 한국어 자연어 이해 우수 |
| 결제       | 토스페이먼츠 SDK      | 카카오/네이버페이 통합 |
| 인증       | NextAuth + 카카오/애플 | 1초 시작 UX |
